Basic push flow
cd /Users/cox/Documents/RobertCox
git status
git add -A
git commit -m "updated"
git push origin main

Check the current remote
git remote -v

Check current branch and changes
git status --short --branch

If GitHub says permission denied to the wrong user
1. Check whether GitHub CLI is installed
   which gh

2. Remove cached GitHub credentials from macOS keychain
   printf "protocol=https\nhost=github.com\n" | git credential-osxkeychain erase

3. If gh is installed, log out and back in
   gh auth logout -h github.com
   gh auth login -h github.com -w

4. When the browser opens, sign in as:
   ccox0719

5. Try the push again
   cd /Users/cox/Documents/RobertCox
   git push origin main

If gh is not installed
1. Set the credential helper
   git config --global credential.helper osxkeychain

2. Remove cached GitHub credentials
   printf "protocol=https\nhost=github.com\n" | git credential-osxkeychain erase

3. Push again
   cd /Users/cox/Documents/RobertCox
   git push origin main

4. Enter:
   Username: ccox0719
   Password: GitHub personal access token
